# General-n Certificate-Bank Mining (2026-07-09)

## Scope and method

This audit asks whether a theorem already proved inside a certificate bank can
be reused uniformly for arbitrary finite ambient cardinality. It covers:

- every bank-like Lean file under `lean/Erdos9796Proof/P97`;
- both blueprint-skipped live trees, `EndpointCertificate` and
  `SurplusCertificate`;
- the multi-center certificate stack;
- the erased-pin ordered/fixed-seed and other live bank modules;
- every raw certificate JSON artifact, including payloads without a live Lean
  path or schema annotation; and
- every Lean file under `attic/`, including the 820 MiB legacy certificate
  tree outside the Lake build; and
- every P97 Lean module in the sibling `p97-rvol` repo, including its unimported
  U1 source-unit certificates and generated U5 incidence incompatibility bank.

The reproducible source census is generated by
`scripts/general-n-bank-miner.py` and stored in:

- `certificates/general_n_bank_mining.json`;
- `certificates/general_n_bank_mining.md`;
- `scripts/p97-rvol-general-n-miner.py`;
- `certificates/p97_rvol_general_n_mining.json`; and
- `certificates/p97_rvol_general_n_mining.md`.

The miner distinguishes ambient-cardinality parameters from polynomial
variable indices such as `nu : Nat -> Real`. It also separates aggregate
geometry consumers, abstract shadow theorems, and fixed-row certificate
expansions. The strongest
aggregate candidates were then checked against their Lean statements, their
actual consumers, the live proof obligations, and `#print axioms` through
`proof-blueprint axioms`.

The source scan is exhaustive over the files and declarations in scope. The
semantic conclusion is conservative: it rules out a directly stated hidden
general-n closer, but it does not claim that no future non-obvious combination
of fixed-row lemmas could contribute to a new proof.

The sibling census is reported separately from the 12,922-file canonical/attic
count below because `p97-rvol` is a second repository with a different import
root and a partially stale extension layer.

## Attic result

The legacy attic tree contributes 646,074 definitions and 340,267 fixed-row
certificate declarations. It has no declaration classified as ambient-n,
ambient-set-uniform, profile-uniform, or aggregate local-geometry-uniform.

This is consistent with its directory structure: it contains old expanded
coefficient terms, generator-zero shards, exact-mask rows, and fixed pattern
coordinators that were replaced by the compressed live checker layout. It does
not contain a forgotten aggregate bank consumer or a theorem quantifying over
an arbitrary finite configuration.

`attic/U2OppCap2Escape.lean` is not a certificate bank. It has two small generic
Finset helpers, but also two residual `sorry`s, and neither helper supplies a
new producer for the current certificate leaves.

Conclusion: there is nothing in the attic certificate bank to promote as a
new general-n theorem.

## Raw artifact result

The raw pass covers 345 certificate JSON files totaling 1.03 GiB, excluding
the generated mining reports themselves. Five files carry ambient-`n`
metadata. All five have a live Lean path or schema annotation, and all belong
to the multi-center `n = 12, ..., 32` inventory/certificate stack discussed
below. There is no un-emitted raw certificate covering arbitrary `n`.

There are 88 JSON files without a live Lean path/schema annotation. Manual
provenance review separates them as follows:

- 70 valid grouped relaxed-split coefficient certificates retained as the
  support-mining bank rather than the live singleton ingestion bank;
- one endpoint core-support census, which is coefficient post-processing and
  not a metric or incidence certificate; and
- 17 generated-input or diagnostic artifacts. Their usable claims are either
  already emitted in `ErasedPinCountRows`, `ErasedPinOrderedProducer`,
  `ErasedPinFixedSeedDFS`, or `SurplusCOMPGBank`, or were refined into the live
  135-row singleton certificate bank.

Thus absence of a path/schema annotation is not itself evidence that an
artifact is unmined. The grouped relaxed-split payloads are the sole genuine
non-ingested coefficient bank, and they are assessed separately below.

The K4 bank was the one unimported bank missing an immediate semantic
consumer. This audit added:

```lean
Problem97.EndpointCertificate.K4WindowBank
  .false_of_mem_k4DeadCertificates
```

For any real valuation and any certificate in the ten-entry dead K4 bank,
zero evaluation of every generator now implies `False`. The theorem is
ambient-cardinality independent and kernel-checks with the approved closure:

```text
propext, Classical.choice, Quot.sound, Lean.ofReduceBool, Lean.trustCompiler
```

This is the algebraic consumer intended for the Candidate-C head split of
`DoubleApexOffSurplusSharedRadiusPairCard12Up`. It does not classify a geometric
four-point window as one of the ten dead classes; that finite/geometric bridge
remains the consumer-side obligation. The module remains intentionally
unimported until that branch uses it.

## Endpoint bank

The strongest hidden aggregate theorem is:

```lean
Problem97.EndpointCertificate.Variables
  .false_of_endpointShadowInBank_of_metricShadow
```

It is genuinely independent of ambient cardinality: for any point assignment,
escape label, and finite endpoint shadow, bank membership plus
`EndpointMetricShadow` implies `False`. Its kernel closure is:

```text
propext, Classical.choice, Quot.sound, Lean.ofReduceBool, Lean.trustCompiler
```

with `Lean.trustCompiler` approved by project policy. No `sorryAx` occurs.

The arbitrary-ambient-set bridge
`false_of_endpointShadowInBank_of_selectedClass_interfaces` and the left/right
`endpoint*_residual_exists_false_of_*` families are also valid for arbitrary
`A : Finset Real^2`; none assumes a fixed value of `A.card`.

These theorems are already consumed by `EndpointCertificate.ResidualSoundness`
and `EndpointCertificate.ResidualCoreData`, and the resulting handoff is already
wired into `Problem97.isM44EndpointResidualsExcluded`. The two live endpoint
holes are exactly the missing producers of:

```text
exists shadow,
  endpointShadowInBank xLabel shadow = true and
  EndpointMetricShadow pointOf shadow
```

The bank does not prove this existence statement. In particular,
`endpointMetricShadow_shadowOfPointClasses_of_sameRadius` can prove the metric
half after a suitable same-radius center-class construction, but it does not
prove that the induced shadow lies in the 117-row endpoint bank. The previously
audited 20/32 payload-mask coverage gap remains outside the certificate
consumer.

Conclusion: the endpoint bank already gives the desired general-n
contradiction theorem, but no unmined theorem supplies the geometric
confinement/bank-membership producer needed to use it.

## Surplus bank

The strongest hidden aggregate theorem is:

```lean
Problem97.SurplusCertificate.RelaxedSplit.Bank.RowZeros
  .false_of_shadowInBank_of_metricShadow
```

It says that exact pinned-bank membership plus a metric interpretation of the
matched point-class shadow implies `False`, with no ambient-cardinality
parameter. Its kernel closure is the same approved native-decision closure as
the endpoint theorem and contains no `sorryAx`.

This theorem is already consumed on both orientations in
`RemovableVertexAxiom/PinnedSurplusBank.lean`. The remaining pinned-surplus
obligation is in the geometric producer layer, currently
`isM44PinnedSurplusNonVExactShapeProducer`; re-mining the row-zero bank cannot
remove that obligation.

The remaining 41 surplus candidates are reusable valuation-soundness and
metric/row-zero helper schemas. They derive generator zeros from supplied
exact-row mask membership or
common-mask membership. They do not prove the missing mask incidences,
confinement, or exact-shape producer.

Conclusion: the surplus bank's general-n consumer is already wired. No hidden
bank theorem closes the producer.

Across the 151 normalized candidate shapes in the two blueprint-skipped banks,
zero statement mentions `OneSidedSeedCandidateRemainder`, `ErasedPin`,
`HasNEquidistantProperty`, or `ProfileStepExtends`. This exact-type cross-check
agrees with the manual review: the skipped declarations implement local
certificate soundness and geometry transport, not any of the current global or
incidence producers.

## Non-ingested grouped relaxed bank

The 70 valid payloads in `certificates/surplus/relaxed_split/` deserve a more
precise conclusion than "superseded by the singleton bank." The singleton bank
refines them, but a grouped payload can require only equations common to
several exact completions. The reproducible census is generated by
`scripts/grouped-relaxed-bank-miner.py` and stored in:

- `certificates/grouped_relaxed_general_n_mining.json`;
- `certificates/grouped_relaxed_general_n_mining.md`.

Of the 70 payloads, 34 cover between two and nine exact pids. Each supports a
potential ambient-cardinality-independent local theorem of the form:

```text
exact v/w equations
  + common erased-center submask equations
  + separator equation (when present)
  -> False
```

The split path identifies the grouped leaf and its common masks; it is not an
additional polynomial equation in the certificate.

These are genuine stronger theorem candidates, not new kernel theorems yet.
They still require Lean emission/checking and a geometric producer for the
listed equations. Twenty-three candidates are under 1 MiB, eight are between
1 and 50 MiB, and three are at least 50 MiB. The 23 small candidates are the
practical first ingestion tranche if the pinned-surplus producer can expose
their common-mask hypotheses.

This bank does not bypass the active erased-pin P2 producer. The six own-kind
private masks in that goal occur in both oriented terminal rows, for 12 census
occurrences total. Every occurrence has zero pinned-bank completions. Hence no
grouped pinned-bank certificate can apply through the existing completion
bridge to those six seeds. The fixed-seed DFS contradiction remains the right
finite consumer after P2 supplies the geometric candidate remainder.

## Erased-pin abstract bank

The expanded classifier finds 26 abstract-shadow contradiction theorems in
`ErasedPinOrderedProducer.lean` and 72 in `ErasedPinFixedSeedDFS.lean`. The
strongest reusable finite conclusion is:

```lean
Problem97.SurplusCOMPGBank
  .false_of_isValidOneSidedSeedRelaxedShapeShadow_of_mem_erasedPinFixedSeed
```

It is independent of ambient cardinality and is already lifted to arbitrary
point-class types by:

```lean
Problem97.SurplusCOMPGBank
  .false_of_erasedPinFixedSeedRelaxedShape_pointClasses
```

The latter is consumed in `SurplusCOMPGBankGeometry.lean`. It requires the
exact opposite-cap masks, the seed's private mask, four-label/no-self shape for
the remaining classes, no-three, prefix-count, and separation facts. The P2
holes are precisely the geometric production of those bundled
`OneSidedSeedCandidateRemainder` facts. The ordered producer's 26 exact-shadow
row theorems are subsumed for the live route by this fixed-seed relaxed-shape
consumer.

## Multi-center bank

The multi-center stack proves
`ProfileClassIncidence.completesL2FullFrom_representativeSurface`, and its
kernel closure is clean under the same approved native-decision policy. The
statement quantifies over `n >= 12`, but its `profileOccursAtN` predicate is
defined by membership in the finite generated `ProfileInventory.rows` bank.
For unscanned `n`, the premise is simply unavailable. It is therefore not a
general-n geometric realization theorem.

The genuinely general-n results remain the abstract theorems in
`MultiCenter/GeneralN.lean` and `MultiCenter/Realization.lean`. To turn the
finite certificate bank into a new general-n theorem, the project still needs
a concrete geometric `ProfileRealizationSurface` together with:

1. extraction and local admissibility;
2. `ProfileStepExtends` or an equivalent persistence theorem;
3. geometric GLOBAL soundness/completion when a GLOBAL conclusion is needed;
4. an explicit consumer on the U1 large-cardinality tail.

The finite representative-surface certificates already eliminate class-bank
enumeration as the missing step. The missing step is geometric realization and
persistence.

### U5 incidence bank

The largest genuinely missed theorem bank is the 296-module
`U5GlobalIncidence...` family. Only three of those module basenames occur here.
After collapsing scalar/metric/class layers, the novel contradiction surface
contains:

- 96 class-level, arbitrary-`CounterexampleData` incidence incompatibility
  consumers; and
- 88 standalone metric-point incompatibility kernels; and
- 112 public scalar-algebra incompatibility kernels supporting those and
  related class patterns.

There are another 133 novel U5 contradiction consumers in 48 non-global-
incidence U5 modules. These are general-n in the relevant sense: their
statements constrain a bounded named point/class packet but do not constrain
the ambient cardinality.

A representative class theorem was rebuilt against the current canonical
dependency:

```lean
Problem97.U5QCriticalTripleClass
  .qcritical_exact_selected_right_qcritical_aux_exact_selected_outer_qcritical_incompatibility
```

The fresh build passes, and its live kernel closure is exactly:

```text
propext, Classical.choice, Quot.sound
```

The bank therefore contains real reusable general-n consumers, not merely old
solver outputs. It still does not contain a theorem classifying an arbitrary
live U1/U5 packet into one of the incompatible incidence families. The missing
step is the same kind of finite/geometric producer seen in the endpoint and
surplus banks.

### Unimported U1 source-unit bank

The 32 files outside the `RVOL` import closure include eight generated
source-unit certificate modules at ordinals
`2, 12, 40, 47, 88, 136, 169, 206`. They expose 18 novel general-n
contradiction consumers:

- eight aggregate `false_of_row_slot_packet` theorems; and
- ten branch-level `false_of_all8_slot_metric_core` theorems.

Each theorem is uniform in `D : CounterexampleData`; only the bounded local row
labels and equal-distance hypotheses are finite. Ordinal 002 was rebuilt from
source against the current dependency. Its aggregate theorem has kernel
closure:

```text
propext, Classical.choice, Quot.sound, Lean.ofReduceBool, Lean.trustCompiler
```

which is accepted by this repo's existing native-decision trust policy. These
18 consumers are immediately portable in principle because the canonical repo
already has `RowSlotLabelPacket`, `AllEightSlotMetricCore`, and the source-unit
checker interface. They do not provide the row-slot placement/equality producer
or prove that the live branch belongs to one of the eight certified ordinals.

The nineteenth unimported contradiction is the standalone, kernel-clean
five-point theorem:

```lean
Problem97.U1LargeCapRouteBTailMetricResidualTarget
  .u1TwoLargeCapObstruction
```

It derives `False` from seven distance equalities and one inequality, with only
the three standard core axioms. It is not a uniform two-large-cap closure: the
stored deterministic pattern-cover samples have 3,713 misses among 4,000
constrained cubes, so this exact five-point pattern occurs in only 287 sampled
cubes. A completeness/producer theorem remains necessary.

### Stale C5D3B surface

The source census also finds hundreds of generic packet adapters and 39 novel
contradiction-shaped declarations in the large C5D3B family. Those cannot yet
be counted as portable kernel results. A fresh minimal build fails in
`RVOL.P97.CGN.CGN4gExt`: `StrictCapBlockData` and nine related declarations are
now already present in canonical `Erdos9796Proof.P97.CGN.CGN4g`.

Thus the old C5D3B oleans are stale with respect to the current dependency.
The extension must first delete/rebase the duplicate declarations; then its
candidate theorem closures must be re-audited. The generated census preserves
them as source candidates but the closure analysis does not rely on them.

## Closure impact

No newly discovered kernel theorem closes a current `sorry`. The mining pass
does identify 34 stronger, non-ingested grouped theorem candidates, 296 new U5
global-incidence contradiction consumers across 96 class-level families, and
19 unimported U1 contradiction consumers. None supplies its own geometric
incidence/placement producer. The
grouped surplus completion bridge also does not cover the six canonical
erased-pin P2 seeds. For the live branches the pass confirms the structural
fact:

```text
finite bank + metric/confinement producer -> general-n contradiction
```

The finite-bank side of that implication is already proved and wired for both
endpoint and surplus banks. The remaining work is entirely on the producer
side:

- endpoint: construct a banked metric shadow for each endpoint residual;
- pinned surplus: finish the non-`.v` exact-shape/metric-shadow producer;
- erased pin: prove the per-center mask-incidence facts yielding the two
  own-kind `OneSidedSeedCandidateRemainder`s on each orientation;
- U1 DoubleApex Candidate C: classify the geometric four-point window into the
  ten-entry dead K4 bank, then apply the new semantic consumer;
- U1 row-slot route: classify the live row packet into one of the eight newly
  mined source-unit ordinals and supply the certificate's explicit hrows;
- U5 incidence route: classify the live bounded class packet into one of the
  96 incompatible class families; and
- multi-center/U1 tail: instantiate geometric realization and persistence.

Neither the attic nor `p97-rvol` changes this producer boundary. The sibling
banks substantially enlarge the available general-n consumer library and may
shorten the U1/U5 finite side, but they do not change the erased-pin producer
boundary. The grouped bank may reduce the pinned-surplus producer if its
common-mask hypotheses are geometrically available.

spines report:

Both targets reach the same five `sorry`-backed project symbols; the additional
